debanding（去色带）滤镜

仅记录在Mangameeya中能成功加载应用的滤镜

banding的产生

色带是在平滑的渐变区域，由于颜色变化小，导致用于量化这段颜色的编码值数量不够，从而在显示器上把渐变反映成了色阶的样子。

参数项 默认值 说明
range 15 色带检测范围
Y Cb Cr 64 各平面色带检测阈值，如当前像素和参考像素之间的差值小于阈值，就会识别为色带
grainY grainC 64 指定在最后去色带阶段添加的微粒(grain)值
sample_mode 2 指定参考像素：模式 1：2像素；模式 2：4像素。
seed Seed for random number generation
blur_first true true: 当前像素与所有像素的平均值进行比较
false: 当前像素与所有像素进行比较 (只有当所有的差值小于阈值时，才被认为是色带)
dynamic_grain false Use different grain pattern for each frame
opt -1 指定优化模式
-1: 使用主机CPU支持的最高等级优化
0: 不优化 （仅用于测试）
1: SSE2 (Pentium 4, AMD K8)
2: SSSE3 (Core 2)
3: SSE4.1 (Core 2 45nm)
mt true 多线程处理，仅支持AviSynth+，true: Y U V 3个平面同时处理提升速度
dither_algo 3 output bits = 16该参数会被忽略
1: 不抖动，LSB is truncated（适合低码率编码）
2: Ordered dithering （有序抖动） （适合低码率编码）
3: Floyd-Steinberg dithering （效果最佳，但容易被x264编码破坏）
keep_tv_range false true: 所有像素会被定位到TV range （luma: 16 ~ 235, chroma: 16 ~ 240）
true适合TV range的视频，除非抖动后像素值会 overflow/underflow
input_mode 0 指定视频源输入模式
0: 8-bit 视频
1: 9-16 bit 视频，逐行扫描（stacked）
2: 9-16 bit 视频，隔行扫描 （interleaved）
input_depth 8 指定视频源输入位深 （范围：8-16）
8: (input_mode = 0)
9-16: (input_mode = 1/2)
output_mode 0 指定视频源输出模式
当指定为2时：帧宽度x2且预览乱码，但x264高位深编码后会恢复正常
output_depth 8 指定视频源输出位深 （范围：8-16）
若设置为16，dither_algo不启用
8: (output_mode = 0)
9-16: (output_mode = 1/2)
random_algo_ref random_algo_grain 1 Choose random number algorithm for reference positions / grains
0: 旧版本算法
1: 均匀分布
2: 高斯分布
random_param_ref random_param_grain 1.0 各随机数生成器参数（目前只对高斯生成器生效）
preset 预设。预设优先级最高 (可组合使用 例：medium/nograin)
depth: y=0/cb=0/cr=0/grainy=0/grainc=0
low: y=32/cb=32/cr=32/grainy=32/grainc=32
medium: y=48/cb=48/cr=48/grainy=48/grainc=48
high: y=64/cb=64/cr=64/grainy=64/grainc=64
veryhigh: y=80/cb=80/cr=80/grainy=80/grainc=80
nograin: grainy=0/grainc=0
luma: cb=0/cr=0/grainc=0
chroma: y=0/grainy=0
medium/nograin: y=48/cb=48/cr=48/grainy=0/grainc=0

注意事项

1.默认设置下Gradfun2db速度比f3kdb快不少
